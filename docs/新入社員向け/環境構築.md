# 新入社員向け環境構築

## この手順のゴール

この手順を最後まで行うと、社員管理アプリを自分のPCで開けるようになります。

利用するものは次の3つです。

- WSL2・Ubuntu: Windowsの中でLinuxを動かす
- Docker Desktop: PHP、MySQLなどの実行環境をまとめて動かす
- Laravel Sail: Docker上のLaravelを簡単なコマンドで操作する

PHP、Composer、Node.js、MySQLをWindowsへ個別にインストールする必要はありません。

> コマンドを入力する画面には「管理者PowerShell」「Ubuntu」の2種類があります。各手順の表記を確認してください。

## 事前に確認すること

- Windows 11を使用している
- インターネットへ接続できる
- PCへソフトウェアをインストールできる
- 研修用GitリポジトリのURLが分かる
- 作業中のファイルを保存し、PCを再起動できる

権限やリポジトリURLが分からない場合は、作業を始める前に研修担当者へ確認してください。

## 1. WSL2とUbuntuを入れる

### 1-1. 管理者PowerShellを開く

1. Windowsのスタートボタンを押す
2. `PowerShell` と入力する
3. 「Windows PowerShell」を右クリックする
4. 「管理者として実行」を選ぶ
5. 確認画面が表示されたら「はい」を押す

### 1-2. Ubuntuをインストールする

管理者PowerShellへ入力します。

```powershell
wsl --install -d Ubuntu
```

インストール後に再起動を求められた場合は、PCを再起動します。

### 1-3. Ubuntuの初期設定をする

1. スタートメニューから「Ubuntu」を開く
2. Linux用のユーザー名を半角英数字で入力する
3. Linux用のパスワードを入力する

パスワード入力中は、画面に文字や記号が表示されません。故障ではありません。入力後にEnterを押します。

### 1-4. WSL2を確認する

管理者PowerShellで実行します。

```powershell
wsl --list --verbose
```

次のようにUbuntuのVERSIONが `2` なら成功です。

```text
NAME      STATE    VERSION
Ubuntu    Stopped  2
```

VERSIONが1の場合は実行します。

```powershell
wsl --set-version Ubuntu 2
```

## 2. Docker Desktopを入れる

### 2-1. インストールする

研修担当者から案内されたDocker Desktopのインストーラー、または[Docker公式手順](https://docs.docker.com/desktop/setup/install/windows-install/)を利用します。

インストール画面では、WSL2を使用する設定を選びます。完了後にPCの再起動を求められた場合は再起動します。

### 2-2. Ubuntu連携を有効にする

1. Docker Desktopを起動する
2. 歯車アイコンの「Settings」を開く
3. 「General」で `Use the WSL 2 based engine` を有効にする
4. 「Resources」→「WSL Integration」を開く
5. `Ubuntu` を有効にする
6. 「Apply & restart」を押す

Docker Desktopの画面でエンジンが起動するまで待ちます。

### 2-3. Dockerを確認する

Ubuntuを開き、実行します。

```bash
docker --version
docker compose version
```

両方にバージョン番号が表示されれば成功です。

`Cannot connect to the Docker daemon` と表示された場合は、Docker Desktopが起動しているか、WSL IntegrationでUbuntuが有効か確認します。

## 3. Gitを確認する

Ubuntuで実行します。

```bash
git --version
```

バージョン番号が表示されない場合は、Ubuntuでインストールします。

```bash
sudo apt update
sudo apt install -y git
```

`sudo`のパスワードは、Ubuntuの初期設定で決めたパスワードです。

## 4. アプリを取得する

Ubuntuで作業用フォルダを作ります。

```bash
mkdir -p ~/work
cd ~/work
```

研修担当者から案内されたURLを使ってcloneします。

```bash
git clone <研修用リポジトリURL> training-app-2026
cd training-app-2026
```

`<研修用リポジトリURL>` は、その文字のまま入力せず、実際のURLへ置き換えます。

現在位置を確認します。

```bash
pwd
ls
```

`pwd` の結果が `/home/自分のユーザー名/work/training-app-2026` に近い形で、`artisan`、`composer.json`、`compose.yaml` が表示されれば成功です。

## 5. 初回セットアップを実行する

プロジェクトの中にいることを確認してから、Ubuntuで実行します。

```bash
bash scripts/sail-setup.sh
```

初回はDocker imageやライブラリを取得するため、時間がかかります。処理中はUbuntuを閉じたり、PCをスリープさせたりしないでください。

このコマンドは次を自動実行します。

1. Sail用の `.env` を作成
2. PHP 8.3のDocker imageを作成
3. Composerライブラリをインストール
4. Laravelのアプリケーションキーを生成
5. LaravelとMySQLを起動
6. テーブルと研修データを作成
7. Bootstrapなどの画面ファイルをbuild

最後に次が表示されれば完了です。

```text
Sail setup completed: http://localhost:8080
```

## 6. 起動状態を確認する

Ubuntuで実行します。

```bash
./vendor/bin/sail ps
```

`laravel.test` と `mysql` が起動状態なら成功です。

テストも実行します。

```bash
./vendor/bin/sail artisan test
```

`Tests: 13 passed` と表示されれば正常です。テスト数はアプリの更新により増える場合があります。

## 7. ブラウザで確認する

[http://localhost:8080](http://localhost:8080) を開きます。

初期ログイン情報は次のとおりです。

```text
メールアドレス: test@example.com
パスワード: password
```

ログイン後、社員一覧が表示されれば環境構築完了です。

## 8. 完了チェック

- [ ] Ubuntuを起動できる
- [ ] `wsl --list --verbose` でUbuntuがVERSION 2になっている
- [ ] Docker Desktopが起動している
- [ ] Ubuntuで `docker compose version` が表示される
- [ ] リポジトリをcloneできた
- [ ] `bash scripts/sail-setup.sh` が完了した
- [ ] `./vendor/bin/sail ps` でLaravelとMySQLが起動している
- [ ] `./vendor/bin/sail artisan test` が成功する
- [ ] ブラウザからログインできる

## 9. うまくいかないときの報告方法

エラーが出た場合は、同じコマンドを何度も実行する前に次を記録します。

```text
実行した手順:
実行したコマンド:
表示されたエラー全文:
自分で確認したこと:
Docker Desktopの状態:
```

Ubuntuで次の結果も取得します。

```bash
./vendor/bin/sail ps
docker compose logs --tail=50 laravel.test
docker compose logs --tail=50 mysql
```

パスワード、`.env`の内容、API tokenなどの秘密情報はIssueやチャットへ貼らないでください。

## 10. 次に読む資料

環境構築後は、毎日の操作を [起動・終了手順](./起動・終了手順.md) で確認してください。
