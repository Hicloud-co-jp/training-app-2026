# 新入社員向け環境構築

## この手順のゴール

社員管理アプリを自分のPCで起動できるようにします。新入社員はUbuntuやLaravel Sailを使わず、PowerShellからDocker Composeを操作します。

必要なものは次の2つです。

- Git for Windows：研修アプリを取得する
- Docker Desktop：PHP、Laravel、MySQLをまとめて動かす

PHP、Composer、Node.js、MySQLをWindowsへ個別にインストールする必要はありません。

## 1. GitHubの利用準備

1. 研修担当者から指定されたメールアドレスで[GitHub](https://github.com/)アカウントを準備する
2. GitHubのユーザー名を研修担当者へ伝える
3. Organizationまたは研修用リポジトリの招待を承認する
4. ブラウザで研修用リポジトリを開けることを確認する

Personal Access Tokenが必要な場合は研修担当者の指示に従い、期限と必要最小限の権限を設定します。tokenをGit、Issue、チャットへ貼らないでください。

## 2. Git for Windowsを入れる

[Git for Windows](https://git-scm.com/download/win)をダウンロードし、標準設定でインストールします。

WindowsのスタートメニューからPowerShellを開き、確認します。

```powershell
git --version
```

バージョン番号が表示されれば完了です。

## 3. Docker Desktopを入れる

[Docker公式手順](https://docs.docker.com/desktop/setup/install/windows-install/)または研修担当者から指定されたインストーラーを使用します。

インストール後、Docker Desktopを開き、「Engine running」と表示されるまで待ちます。PowerShellで確認します。

```powershell
docker --version
docker compose version
```

両方にバージョン番号が表示されれば完了です。Docker Desktop内部でWSL2が利用される場合がありますが、新入社員がUbuntuを追加して操作する必要はありません。

## 4. アプリを取得する

PowerShellで実行します。

```powershell
New-Item C:\mywork -ItemType Directory -Force
cd C:\mywork
git clone <研修用リポジトリURL> training-app-2026
cd C:\mywork\training-app-2026
```

`<研修用リポジトリURL>`は、研修担当者から案内された実際のURLへ置き換えます。

## 5. 初回セットアップを実行する

Docker Desktopが起動している状態で、PowerShellから実行します。

```powershell
cd C:\mywork\training-app-2026
.\scripts\docker-setup.bat
```

このバッチはDocker image、PHPパッケージ、DB、研修データ、画面ファイルをまとめて準備します。初回は時間がかかるため、PowerShellを閉じたりPCをスリープさせたりしないでください。

最後に次が表示されれば完了です。

```text
セットアップが完了しました: http://localhost:8080
```

## 6. ブラウザで確認する

[http://localhost:8080](http://localhost:8080)を開きます。

```text
メールアドレス: test@example.com
パスワード: password
```

社員一覧が表示されれば環境構築は完了です。

## 7. うまくいかないとき

PowerShellで次を実行し、結果を研修担当者へ伝えます。

```powershell
docker compose ps
docker compose logs --tail=30 laravel.test
docker compose logs --tail=30 mysql
```

実行したコマンド、エラーメッセージ、上記の結果を共有してください。`.env`、パスワード、tokenは共有しません。
