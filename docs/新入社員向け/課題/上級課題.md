# 上級課題：DB正規化・CSV・画像アップロード

上級は3つの独立課題に分けます。1つずつbranchとPull Requestを作成してください。

## 課題A：部署をマスター化する

### 目的

- データベース設計を理解する
- 正規化の目的と実施方法を学ぶ
- migrationとEloquentのリレーションを学ぶ

### 対象範囲

最初は社員の新規登録画面を対象とします。編集、削除、検索は横展開課題とし、最初の必須範囲には含めません。

ただし、スキーマ変更後に社員一覧がエラーになる場合は、講師のフォローを受けて一覧表示を維持します。

### 進め方

1. 現在の`employees.department`データを調査する
2. 空文字、表記ゆれ、対応不能データを特定する
3. `departments`テーブルをmigrationで作成する
4. 部署の初期データをSeederまたはmigration方針に従って投入する
5. employeesとdepartmentsの関連方法を決める
6. 既存社員データを部署IDへ移行する
7. EmployeeとDepartmentへリレーションを定義する
8. 新規登録画面の部署入力をtextからselectへ変更する
9. 部署マスタから選択肢を取得する
10. 選択された部署で社員を登録する

### 重要な注意

本番相当データをA5:SQL Mk-2等で直接DELETE・UPDATEする前に、必ずバックアップと対象件数を確認します。研修リポジトリでは、誰が実行しても同じ状態になるよう、可能な限りデータ補正もmigrationまたは専用コマンドで再現します。

既存の`department`をVARCHARからBIGINTへ直接変更する方法と、新しい`department_id`を追加して段階移行する方法を比較し、講師と方針を決めてください。推奨は次です。

```text
department_idをnullableで追加
  ↓
既存department文字列からIDを設定
  ↓
画面・Modelをdepartment_idへ切替
  ↓
未移行データがないことを確認
  ↓
旧departmentを削除
```

### 完了条件

- departmentsテーブルがmigrationで作られる
- 既存データの移行方法を説明できる
- 社員登録画面に部署プルダウンが表示される
- 登録した社員と部署が関連付く
- 外部キーとリレーションのテストがある
- 一覧画面がエラーなく表示される

## 課題B：社員データをCSV出力する

### 目的

- SQLのWHERE条件を利用する
- 画面の検索条件と出力条件を一致させる
- ファイル生成とダウンロードを理解する
- Flysystemを利用する

### Git運用

mainから課題用branchを作成します。Issue番号が確定している場合は、指定された番号をbranch名に含めます。

### 画面仕様

- 社員一覧へ「CSVダウンロード」ボタンを追加する
- 現在の検索条件に一致する社員だけを出力する
- 社員番号の昇順で出力する

### CSV仕様

| 項目 | 仕様 |
| --- | --- |
| ヘッダー | 出力対象の全カラム名 |
| 文字コード | Shift-JIS |
| 改行 | CRLF |
| ファイル名 | `YYYYMMDD-HHmmss.csv` |
| 並び順 | 社員番号の昇順 |
| エスケープ | カンマ、ダブルクォート、改行を考慮 |

`deleted_at`や内部管理項目を「employeeの全カラム」に含めるかは、個人情報と利用目的を確認して講師が開始前に確定します。

### ファイル管理

- LaravelのStorage・Flysystemを使う
- `storage/app/temp`へ生成する
- この課題では自動削除を必須にしない
- 生成後にHTTPダウンロードする
- ファイルパスを利用者入力から組み立てない

### テスト

- 検索条件に一致する社員だけが含まれる
- 社員番号順である
- ヘッダーがある
- カンマ、引用符、日本語を含む値を出力できる
- 認証されていない利用者は出力できない

## 課題C：プロフィール画像をアップロードする

### 目的

- ファイルアップロードの処理フローを理解する
- Flysystemのdiskを用途別に設定する
- Bootstrapモーダルを実装する
- 保存先とDBの関連を説明する

### 画面仕様

- 社員新規登録画面の末尾へ画像入力を追加する
- 登録時に画像をローカルstorageへ保存する
- 社員一覧の社員名を押すとモーダルを開く
- モーダル内へ画像を横幅100%で表示する
- 画像がない場合もモーダルを開き、「画像は登録されていません」と表示する
- 画像の更新・削除は対象外

「ユーザー一覧」ではなく、このアプリでは「社員一覧」を対象とします。

### DB・保存仕様

- employeesへ`img_id`を追加する
- `img_id`は推測しにくいUUID等を検討する
- 画像用にCSVとは別のFlysystem diskを設定する
- 保存先はLaravelのstorage配下とする

```text
storage/app/employees/img/{img_id}/
```

ファイル名、拡張子または相対パスをDBへ持つ必要があるかを設計し、`img_id`だけで画像を特定できる方法を説明してください。

### 最低限の安全確認

- 画像ファイルであること
- 許可する拡張子・MIME type
- 最大ファイルサイズ
- 元ファイル名をそのまま保存名に使わない
- public公開する場合のURL生成方法

### テスト

- 画像付きで社員登録できる
- 指定diskへ画像が保存される
- 画像なしでも社員登録できる
- 不正なファイルを拒否する
- 一覧または表示用レスポンスから画像を確認できる
