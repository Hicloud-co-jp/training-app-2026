# 初級課題：社員一覧を使いやすくする

## 目的

既存の社員一覧を改修し、日付計算、URLパラメータ、並び替え、入力検証を学びます。

## 前提

- 社員一覧・登録・編集を操作できる
- Route、Controller、Model、Viewの役割を確認済み
- Gitで課題用branchを作成できる

## 課題1：入社年数を表示する

社員一覧へ、入社日から現在までの経過年数を表示してください。

### 学習ポイント

- Laravelでの日付の扱い
- Carbonを使った日付計算
- Modelのcast
- ControllerからViewへ渡されたデータの表示
- 入社日が未登録の場合の扱い

### 完了条件

- 入社日がある社員に経過年数が表示される
- 入社日がない社員でもエラーにならない
- 入社日の有無や日付の境界を画面で確認している

表示例は「3年」「1年6か月」などから、着手前に講師と統一します。

## 課題2：検索条件を保持する

社員を検索した後、ページネーションで2ページ目以降へ移動しても検索条件が消えないようにしてください。

### 学習ポイント

- GETリクエスト
- URLクエリパラメータ
- ページネーション
- `withQueryString()` または `appends()` の役割

### 完了条件

- 検索後のURLに検索条件が含まれる
- 2ページ目へ移動しても同じ検索結果が表示される
- 検索解除で条件をリセットできる

## 課題3：一覧をソートする

社員一覧の列ヘッダーをクリックすると、昇順・降順を切り替えられるようにしてください。

対象列は、最初に次から2列以上を選びます。

- 社員番号
- 氏名
- 部署
- 入社日

### 学習ポイント

- `orderBy`
- URLクエリパラメータ
- 昇順・降順の切り替え
- 利用者から渡された列名をそのままSQLへ渡さない安全な設計
- 検索条件とソート条件の同時保持

### 完了条件

- ヘッダークリックで昇順・降順が切り替わる
- 現在の並び順が画面で分かる
- ページ移動後も検索・ソート条件が残る
- 許可していない列名や並び順は拒否または既定値へ戻る

## 課題4：バリデーションを強化する

社員登録・更新の入力値について、文字数と形式を確認してください。

### 対象例

| 項目 | 確認内容 |
| --- | --- |
| 社員番号 | 必須、20文字以内、重複禁止 |
| 氏名 | 必須、100文字以内 |
| 部署 | 任意、100文字以内 |
| 入社日 | 日付形式、未来日不可 |
| メール | メール形式、255文字以内 |

### 学習ポイント

- Form Request
- 登録と更新で共通するルール
- unique検証で更新対象自身を除外する方法
- エラーメッセージの画面表示
- 正常な入力と入力エラーの動作確認

## 提出物

- 改修コード
- 検索・ソート時のURL例
- 変更したファイルと理由を記載したREADMEまたはPull Request
- 正常時・エラー時の動作確認結果

## 本人説明で確認すること

- Carbonをどこで利用したか
- 検索条件がページ移動後も残る理由
- ソート可能な列を制限する理由
- バリデーションをController外へ置く理由
- 画面操作で確認した入力値と結果
