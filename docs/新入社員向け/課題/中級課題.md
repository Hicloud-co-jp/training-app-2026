# 中級課題：郵便番号APIで住所を検索する

## 目的

画面を一から作成し、画面遷移に関係するMVC全体と、外部APIの呼び出し方を理解します。

この課題では、住所の登録や厳格な郵便番号判定には重点を置きません。

## 完成イメージ

社員一覧画面の「社員を登録」ボタンの横に「住所検索」ボタンを追加します。

```text
社員一覧
├─ 社員を登録
└─ 住所検索
      ↓
住所検索画面
├─ 郵便番号入力
├─ 検索ボタン
└─ 検索結果をテキスト表示
```

検索結果は同じ住所検索画面に表示します。DBへの登録処理は行いません。

## 課題1：住所検索画面を作る

- 住所検索用のRouteを追加する
- Controllerを追加する
- Bladeを新規作成する
- 社員一覧から住所検索画面へ遷移できるようにする
- 住所検索画面から社員一覧へ戻れるようにする

見た目は既存のBootstrap画面に合わせます。

## 課題2：郵便番号を検証する

標準のLaravelバリデーションを使い、最低限の確認をします。

例：

- 必須
- 文字列
- `1234567` または `123-4567` の形式

この課題では、実在する郵便番号か、事業所用郵便番号かなどの厳格な判定は不要です。

## 課題3：外部APIを呼び出す

入力された郵便番号を使って、研修で指定された郵便番号検索APIへリクエストしてください。

### 学習ポイント

- APIとは何か
- HTTPリクエストとレスポンス
- Laravel HTTP Client
- JSONの読み取り
- 接続失敗、該当なし、成功の違い
- 外部サービスへ依存する処理の扱い

APIのURLや、必要な場合のAPIキーは設定ファイルまたは環境変数で管理し、Controllerへ直接埋め込まないでください。

## 課題4：同じ画面へ結果を表示する

検索成功後、同じ住所検索画面へ次をテキスト表示してください。

- 郵便番号
- 都道府県
- 市区町村
- 町域
- 結合した住所

表示方法は実装者が決めます。住所登録用フォームを作る必要はありません。

## 必須の状態

| 状態 | 期待する画面 |
| --- | --- |
| 初回表示 | 入力欄だけを表示 |
| 入力エラー | 入力欄とエラーメッセージを表示 |
| 検索成功 | 入力欄と住所結果を表示 |
| 該当なし | 該当しないことを表示 |
| API接続失敗 | 利用者向けエラーを表示し、画面を壊さない |

## テスト

外部APIをテストから直接呼ばず、LaravelのHTTP fakeを使います。

- 検索画面を表示できる
- 郵便番号の入力エラーを表示する
- API成功時に住所を表示する
- 該当なしを表示する
- APIエラー時に適切なメッセージを表示する

## 提出物

- 新規画面、Route、Controller
- API設定
- Feature Test
- 画面遷移図
- APIリクエストから画面表示までの処理フロー
- 利用したAPIのリクエスト・レスポンス例

## 本人説明で確認すること

- 新しく作ったRoute、Controller、Viewの関係
- APIと通常のWeb画面の違い
- JSONのどの値を画面へ表示したか
- 同じ画面で初回・成功・失敗をどう切り替えたか
- テストで実際の外部APIを呼ばない理由
