# 社員管理アプリ 起動・終了手順

この文書は、環境構築が終わった後に毎日使う手順です。コマンドはUbuntuで実行します。
## 最短早見表

```bash
# 起動
cd ~/work/training-app-2026
docker compose up -d
docker compose ps

# 終了
docker compose stop

# ブラウザ
http://localhost:8080
```
## 毎日の起動

### 1. Docker Desktopを起動する

WindowsのスタートメニューからDocker Desktopを起動します。Dockerエンジンの起動が完了するまで待ちます。

### 2. Ubuntuを開く

WindowsのスタートメニューからUbuntuを開きます。

### 3. プロジェクトへ移動する

```bash
cd ~/work/training-app-2026
```

現在位置が分からない場合は確認します。

```bash
pwd
ls
```

`artisan`、`composer.json`、`compose.yaml` があれば正しい場所です。

### 4. Sailを起動する

```bash
./vendor/bin/sail up -d
```

`-d` はバックグラウンドで動かす指定です。Ubuntuで別のコマンドを続けて入力できます。

### 5. 起動状態を確認する

```bash
./vendor/bin/sail ps
```

確認対象は次の2つです。

- `laravel.test`: Laravelアプリ
- `mysql`: データベース

両方が起動していれば正常です。

### 6. ブラウザで開く

[http://localhost:8080](http://localhost:8080) を開きます。

```text
メールアドレス: test@example.com
パスワード: password
```

## 毎日の終了

作業内容を保存し、必要なcommitやpushを終えてから実行します。

```bash
./vendor/bin/sail stop
```

Dockerコンテナを削除して終了する場合は次を使います。

```bash
./vendor/bin/sail down
```

どちらの場合もMySQLのデータは残ります。通常は `stop` で構いません。

最後にDocker Desktopを終了できます。

## よく使うコマンド

```bash
# テスト
./vendor/bin/sail artisan test

# migration状態確認
./vendor/bin/sail artisan migrate:status

# ルート一覧
./vendor/bin/sail artisan route:list

# Laravelのキャッシュ削除
./vendor/bin/sail artisan optimize:clear

# ログ確認
./vendor/bin/sail artisan pail

# 画面開発用Vite
./vendor/bin/sail pnpm run dev
```

`pnpm run dev` は実行中のままになるため、停止するときは `Ctrl+C` を押します。

## ソースを更新した後

最新版を取得します。

```bash
git pull
```

変更内容に応じて次を実行します。全部を毎回実行する必要はありません。

```bash
# composer.jsonまたはcomposer.lockが変わった
./vendor/bin/sail composer install

# package.jsonまたはpnpm-lock.yamlが変わった
./vendor/bin/sail pnpm install --frozen-lockfile

# migrationが増えた
./vendor/bin/sail artisan migrate

# 画面資産をbuildする
./vendor/bin/sail pnpm run build

# 最後にテストする
./vendor/bin/sail artisan test
```

## 起動できないとき

### Dockerへ接続できない

Docker Desktopが起動しているか確認します。

```bash
docker info
```

### Sailコマンドがない

プロジェクトの場所を確認します。

```bash
pwd
ls vendor/bin/sail
```

`vendor/bin/sail` が存在しない場合は、研修担当者へ連絡してください。初回セットアップが完了していない可能性があります。

### 画面が開かない

```bash
./vendor/bin/sail ps
docker compose logs --tail=50 laravel.test
```

### DBエラーが出る

```bash
docker compose logs --tail=50 mysql
./vendor/bin/sail artisan migrate:status
```

### 8080番ポートが使われている

`.env` の次の2項目を変更し、Sailを再起動します。

```env
APP_PORT=8081
APP_URL=http://localhost:8081
```

```bash
./vendor/bin/sail down
./vendor/bin/sail up -d
```

変更後は `http://localhost:8081` を開きます。

## DBデータを消す操作

次のコマンドはMySQLのDocker volumeも削除します。

```bash
./vendor/bin/sail down -v
```

研修データを消してよい場合以外は実行しないでください。分からない場合は研修担当者へ確認します。
