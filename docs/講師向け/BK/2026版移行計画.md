# training-app 2026 移行計画

## 1. 推奨する完成像

2026版も「認証付き社員管理」を題材として維持します。ただし、完成コードを読むだけの教材ではなく、段階ごとに設計判断とテストを学べる構成へ変えます。

残す中核は、社員一覧、検索、登録、更新、論理削除、認証、読み取り API、Factory/Seeder、Feature Test です。刷新する中核は、プロジェクト基盤、入力検証、認可、ログ、API 契約、フロントビルド、CI/CD、セットアップ文書です。

## 2. 移行方式

Laravel の新規スケルトンを作成し、現行コードを機能単位で移植する「再構築方式」を推奨します。現行リポジトリをそのままアップグレードすると、欠落した `package.json`、追跡済み `.env`、古い workflow、Breeze 生成物と独自 UI の混在まで引き継ぐためです。

フレームワークや PHP のバージョンは、実装開始時点で公式にサポートされる組み合わせを確認し、ADR（設計判断記録）へ固定します。「2026だから特定の最新版」と先に決めず、研修期間中のセキュリティサポートと受講者 PC での導入容易性を優先します。

## 3. フェーズ別ロードマップ

### Phase 0: 仕様を固定する

- 研修対象者、Windows 前提の有無、研修日数を決める
- 社員メールの一意性、未来の入社日、削除復元、権限差を決める
- Web と API の学習範囲を決める
- Bootstrap を継続するか、Laravel 標準寄りの UI に統一するか決める
- `docs` と `docks` の正式なディレクトリ名を決める

完了条件は、画面一覧、API 一覧、データ辞書、受け入れ条件が短い文書として合意されていることです。

### Phase 1: 安全な土台を作る

- サポート対象の PHP/Laravel で新規プロジェクトを生成
- `.env.example` のみを管理し、実値を Git に入れない
- 漏えいした可能性のあるアプリキー、DB、AWS、メール等の秘密をローテーション
- `public/phpinfo.php` を持ち込まない
- Composer と Node の manifest/lock を両方そろえる
- `.editorconfig`、Pint、PHPStan/Larastan、PHPUnit を最初に動かす
- CI で install、test、static analysis、format check、asset build を実行する

完了条件は、クリーン環境で README の手順だけを使って構築でき、CI が成功することです。

### Phase 2: ドメインとDBを移植する

- `Employee`、migration、Factory、Seeder を移植
- migration は2026版用に整理し、新規構築時は `deleted_at` を初期定義へ含める
- 検索条件と並び順を仕様化する
- メールの一意性やインデックスを業務仕様に合わせて追加する
- 日付境界とタイムゾーンをテストする

既存データを本当に引き継ぐ場合は、スキーマ移植とは別に、件数照合・NULL・重複メール・文字コード・削除済みデータを検査する移行スクリプトとロールバック手順を用意します。

### Phase 3: Web 機能を移植する

- 認証機能を現行の公式スターター方針に沿って導入
- Employee 用 Form Request を作り、登録・更新の検証を共通化
- Controller は validated data のみを受け渡す
- Employee Policy を導入し、閲覧・作成・更新・削除を明示する
- 一覧の検索条件をページングリンクへ引き継ぐ
- モーダルのアクセシビリティ、キーボード操作、エラー時の再表示を確認する

### Phase 4: API を移植する

- `/api/v1/employees` のように契約の版を明示する
- `search`、`department`、`per_page` を Form Request で検証する
- `per_page` は整数かつ 1〜100 とするなど、異常値の応答を固定する
- JSON の成功・検証エラー・認証エラー・404 の形式を文書化する
- Sanctum token の発行、権限、失効、保管を教材に含める
- 書き込み API は研修目標に必要な場合だけ別課題として追加する

### Phase 5: ログと運用を整える

- 構造化ログの項目を決め、メール、パスワード、token、全リクエスト内容を記録しない
- 例外はフレームワークの標準報告を基礎とし、二重ログを避ける
- `/up` と `/api/ping` の役割を整理する
- 配備は main への merge と品質ゲート成功後だけに限定する
- SSH host key を検証し、失敗時も一時的な通信許可を必ず閉じる
- production の migration、cache、queue、rollback 手順を作る

### Phase 6: 教材として仕上げる

- 導入 → MVC → DB → 検証 → 認証/認可 → API → テスト → 運用の順に教材を並べる
- 各章に目的、事前知識、演習、確認方法、よくある失敗を置く
- 完成版だけでなく、章ごとの開始タグまたはブランチを用意する
- 既存の69テストを移植し、仕様変更部分は期待値を更新する
- 文書中のコマンドをクリーン環境で通し、リンク切れと画面差分を確認する

## 4. 優先順位付きバックログ

| 優先度 | 作業 | 理由 |
| --- | --- | --- |
| P0 | `.env` の追跡解除と秘密のローテーション | 漏えい影響を止める |
| P0 | `public/phpinfo.php` を除外 | 環境情報の公開を防ぐ |
| P0 | 再現可能な PHP/Node 依存定義を作る | 新規受講者が構築できるようにする |
| P0 | CI にテスト・静的解析・ビルドを追加 | 壊れた教材の配布を防ぐ |
| P1 | Form Request と Policy の導入 | 入力境界と権限を学べるようにする |
| P1 | ログの個人情報を最小化 | 教材で安全な実装例を示す |
| P1 | API v1 とエラー契約を定義 | クライアントとの互換性を保つ |
| P1 | セットアップ手順を実機検証 | 文書だけで環境構築可能にする |
| P2 | 削除済み一覧・復元を演習化 | SoftDeletes を最後まで学ぶ |
| P2 | E2E/ブラウザテストを追加 | モーダル操作とJSを保証する |
| P2 | アクセシビリティ確認 | UI品質も研修範囲に含める |

## 5. 移植対象の対応表

| 現行資産 | 2026版での扱い |
| --- | --- |
| `app/Models/Employee.php` | 仕様確認後に移植。型とスコープを整理 |
| Web `EmployeeController` | 処理を移植し、検証は Form Request へ分離 |
| API `EmployeeController` | v1 配下へ移植。クエリ検証を追加 |
| `EmployeeResource` | 契約テストとともに移植。未来日を定義 |
| migration | 新規版として整理。実データ移行とは分離 |
| Factory / Seeder | 固定シナリオとランダムデータを分けて移植 |
| Employee 関連テスト | 先に移植して振る舞いの安全網にする |
| Breeze 生成コード | コピーせず、新版で再生成して差分だけ反映 |
| Blade 社員画面 | UI 方針決定後に移植または再実装 |
| `docs/` | 実装照合後に必要部分を改稿して移す |
| `.github/workflows/main.yml` | 持ち込まず、品質CIと配備を分離して再作成 |
| `.env`, `public/phpinfo.php` | 移植禁止 |

## 6. Definition of Done

2026版の初回リリースは、次をすべて満たした時点で完了とします。

- 新しいPCまたはクリーンな環境で、文書どおりに起動できる
- PHP依存とNode依存をロックファイルから再現できる
- 秘密情報や診断ページがGitと公開ディレクトリに存在しない
- Web の社員 CRUD、検索、ページング、論理削除が動作する
- API の認証、一覧、検索、詳細、エラー形式が仕様どおりである
- 入力検証と認可が Controller 外に明示されている
- テスト、静的解析、フォーマット検査、asset build が CI で成功する
- ログにパスワード、token、社員メール全文を出さない
- デプロイ失敗時の切り戻し手順が検証済みである
- 文書の「実装済み」という記述がコードと一致している

## 7. 最初の実装単位

最初のプルリクエストは業務機能を含めず、「新規 Laravel 基盤、依存関係、秘密情報対策、CI、最小 README」だけにします。次に Employee の migration/model/test、その次に Web、その次に API と分割すると、レビューしやすく、研修教材としても段階を追えます。
